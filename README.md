**Abstract**
This project builds a machine learning system that predicts the likelihood of heart disease in a patient based on 13 clinical features. It uses the UCI Cleveland Heart Disease Dataset (303 patients) and serves predictions through a Flask web application. The final model achieves 88.5% test accuracy and 93.4% ROC-AUC score.

---

**Keywords**
Heart Disease Prediction, Logistic Regression, Scikit-learn, Flask, Machine Learning, Classification, UCI Dataset, StandardScaler, Cross-Validation

---

**I. Introduction**
Heart disease is one of the leading causes of death worldwide. Early detection can save lives. This project uses machine learning to analyze patient health data and predict whether a person has heart disease or not. A doctor or researcher can enter patient values into a simple web form and get an instant prediction with a risk level (Low / Medium / High).

---

**II. Literature Review**
Several studies have used the UCI Cleveland Heart Disease dataset for binary classification. Common approaches include Logistic Regression, Decision Trees, and SVMs. Most prior work focuses only on accuracy. This project improves on that by using cross-validation, AUC scoring, and a proper preprocessing pipeline to avoid data leakage — common mistakes in earlier implementations.

---

**III. Methodology**

**A. Data Collection**
Used the UCI Cleveland Heart Disease Dataset — 303 rows, 13 input features (age, sex, chest pain type, cholesterol, etc.), and 1 binary target (0 = no disease, 1 = disease present).

**B. Data Preprocessing**
Applied StandardScaler to normalize all features. Scaling is done inside a Scikit-learn Pipeline so the scaler fits only on training data, preventing data leakage into the test set.

**C. Model Training**
Three models were trained and compared — Logistic Regression, Random Forest, and SVM. Each was wrapped in a Pipeline with a scaler step. GridSearchCV was used to tune hyperparameters for Logistic Regression.

**D. Evaluation**
Models were evaluated using 5-Fold Cross-Validation, Test Accuracy, ROC-AUC score, Confusion Matrix, Precision, Recall, and F1-Score.

**E. Deployment**
The best model (Logistic Regression) is saved using joblib and served through a Flask web application with a form-based UI for real-time predictions.

---

**IV. Implementation**
- `Heart_Disease_Improved.ipynb` — Full EDA, model training, comparison, and saving
- `app.py` — Flask server with `/predict` and `/model-info` API endpoints
- `templates/index.html` — Web frontend for user input
- `heart_disease_model.pkl` — Saved trained model (auto-generated on first run)
- `static/` — EDA plots generated by the notebook

---

**V. Results and Discussion**

| Metric | Score |
|---|---|
| Test Accuracy | 88.5% |
| ROC-AUC | 93.4% |
| 5-Fold CV Accuracy | 83.8% ± 3.2% |
| Precision (macro) | 88% |
| Recall (macro) | 88% |

Logistic Regression performed best overall — it was accurate, interpretable, and stable across folds. Random Forest showed higher training accuracy but similar test performance. SVM performed competitively but required more tuning.

---

**VI. Limitations**
- Small dataset (only 303 patients) — may not generalize to all populations
- Only the Cleveland subset of the UCI dataset is used
- No deep learning or ensemble methods explored
- Model is not validated on real hospital data
- Web app has no authentication or data storage
